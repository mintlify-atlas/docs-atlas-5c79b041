## What is pin mapping?

Pin mapping connects the physical GPIO pins on your RP2040 board (labeled GP0 through GP29) to logical gamepad actions. Each pin can be assigned one action â€” a button press, a D-pad direction, an analog axis direction, a sustain mode, a macro trigger, or a special function.

The pin mapping page shows a grid of all available GPIO pins. For each pin, you select one or more actions from a grouped dropdown.

<Warning>
  Mapping a pin to a function that is not physically connected can leave the device in a non-functional state. If that happens, go to the **Reset Settings** page to clear the invalid configuration.
</Warning>

## Profiles

GP2040-CE supports up to **6 independent profiles**, each with its own complete pin map and a custom label (up to 16 alphanumeric characters). Profiles let you switch button layouts on the fly using hotkeys without replugging the device.

- **Profile 1** is the base profile and is always enabled.
- **Profiles 2â€“6** can be enabled or disabled individually. The active profile cannot be disabled.
- The **Copy Base Profile** button copies Profile 1's pin assignments into the current profile as a starting point.
- Use the **Capture** button to physically press a button and automatically detect which GPIO pin it is connected to.

<Info>
  The active profile is selected in the **Settings** page under the Profile dropdown. Profiles can also be switched at runtime using hotkey combos.
</Info>

## GPIO actions (GpioAction)

The following actions can be assigned to any GPIO pin. Actions are grouped into **Buttons** (standard button and D-pad presses) and **Actions** (special functions).

### Special values

| Action | Description |
|---|---|
| `NONE` | Pin is unused. Default for unassigned pins. |
| `ASSIGNED_TO_ADDON` | Pin is controlled by an add-on (e.g. analog stick, SNES adapter). Cannot be changed here. |
| `RESERVED` | Pin is reserved by the firmware or hardware and cannot be assigned. |
| `CUSTOM_BUTTON_COMBO` | Pin triggers a combination of multiple button actions simultaneously. |

## Configuring pin mappings

<Steps>
  <Step title="Select a profile">
    Choose a profile tab from the left sidebar. Profile 1 is the base profile; profiles 2â€“6 require enabling before they can be saved.
  </Step>
  <Step title="Assign actions to pins">
    For each GPIO pin (GP0â€“GP29), use the dropdown to assign one or more actions. Buttons and D-pad actions can be combined on a single pin using the multi-select. Actions (mode sustains, navigation) are single-select only.
  </Step>
  <Step title="Use the Capture button (optional)">
    Click **Capture** and press a physical button on the device to automatically identify which GPIO pin it is wired to.
  </Step>
  <Step title="Save">
    Click **Save** to write the profile's pin map to flash memory.
  </Step>
</Steps>

<Warning>
  Do not assign the same action to two different pins within the same profile. Duplicate assignments can cause unexpected behavior such as stuck inputs.
</Warning>
