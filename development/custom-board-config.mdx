title: Custom board config
description: How to create a custom board configuration for a new RP2040-based controller by defining GPIO pin assignments, display settings, and LED configuration.
---

A board config is a collection of files inside the `configs/` directory that tells the GP2040-CE firmware how to map GPIO pins to buttons, configure hardware peripherals, and set default options for a specific controller board. Each board has its own subdirectory.

## Directory structure

```
configs/
├── Pico/               # Raspberry Pi Pico reference config
│   ├── BoardConfig.h   # Pin assignments and hardware defines
│   ├── CMakeLists.txt  # Board-level CMake (usually stubs)
│   └── README.md       # Board-specific notes
├── Blank/              # Minimal template for new boards
│   ├── BoardConfig.h
│   ├── CMakeLists.txt
│   └── README.md
└── <YourBoard>/        # Your custom config goes here
```

The `GP2040_BOARDCONFIG` CMake variable (or environment variable) selects which subdirectory to use at build time. The build system looks for `configs/<GP2040_BOARDCONFIG>/<GP2040_BOARDCONFIG>.cmake` and includes it if present.

## Files in a board config

### `BoardConfig.h`

This is the primary configuration header. It defines all GPIO pin assignments, peripheral settings, and default firmware behavior using C preprocessor macros.

The `Blank/BoardConfig.h` is the minimal starting point — it contains only keyboard key mappings and no GPIO assignments, so the board boots with no buttons connected:

```c
// configs/Blank/BoardConfig.h
#ifndef PICO_BOARD_CONFIG_H_
#define PICO_BOARD_CONFIG_H_

#include "enums.pb.h"
#include "class/hid/hid.h"

#define BOARD_CONFIG_LABEL "Blank Configuration"

// Keyboard Mapping Configuration
//                                            // GP2040 | Xinput | Switch  | PS3/4/5  | Dinput | Arcade |
#define KEY_DPAD_UP     HID_KEY_ARROW_UP      // UP     | UP     | UP      | UP       | UP     | UP     |
#define KEY_DPAD_DOWN   HID_KEY_ARROW_DOWN    // DOWN   | DOWN   | DOWN    | DOWN     | DOWN   | DOWN   |
#define KEY_DPAD_RIGHT  HID_KEY_ARROW_RIGHT   // RIGHT  | RIGHT  | RIGHT   | RIGHT    | RIGHT  | RIGHT  |
#define KEY_DPAD_LEFT   HID_KEY_ARROW_LEFT    // LEFT   | LEFT   | LEFT    | LEFT     | LEFT   | LEFT   |
#define KEY_BUTTON_B1   HID_KEY_SHIFT_LEFT    // B1     | A      | B       | Cross    | 2      | K1     |
#define KEY_BUTTON_B2   HID_KEY_Z             // B2     | B      | A       | Circle   | 3      | K2     |
#define KEY_BUTTON_R2   HID_KEY_X             // R2     | RT     | ZR      | R2       | 8      | K3     |
#define KEY_BUTTON_L2   HID_KEY_V             // L2     | LT     | ZL      | L2       | 7      | K4     |
#define KEY_BUTTON_B3   HID_KEY_CONTROL_LEFT  // B3     | X      | Y       | Square   | 1      | P1     |
#define KEY_BUTTON_B4   HID_KEY_ALT_LEFT      // B4     | Y      | X       | Triangle | 4      | P2     |
#define KEY_BUTTON_R1   HID_KEY_SPACE         // R1     | RB     | R       | R1       | 6      | P3     |
#define KEY_BUTTON_L1   HID_KEY_C             // L1     | LB     | L       | L1       | 5      | P4     |
#define KEY_BUTTON_S1   HID_KEY_5             // S1     | Back   | Minus   | Select   | 9      | Coin   |
#define KEY_BUTTON_S2   HID_KEY_1             // S2     | Start  | Plus    | Start    | 10     | Start  |
#define KEY_BUTTON_L3   HID_KEY_EQUAL         // L3     | LS     | LS      | L3       | 11     | LS     |
#define KEY_BUTTON_R3   HID_KEY_MINUS         // R3     | RS     | RS      | R3       | 12     | RS     |
#define KEY_BUTTON_A1   HID_KEY_9             // A1     | Guide  | Home    | PS       | 13     | ~      |
#define KEY_BUTTON_A2   HID_KEY_F2            // A2     | ~      | Capture | ~        | 14     | ~      |
#define KEY_BUTTON_FN   -1                    // Hotkey Function                                        |

#endif
```

### `CMakeLists.txt`

This file is included by the build system but is typically left as stubs for most boards. It exists mainly as a placeholder for any board-specific CMake customization.

```cmake
# configs/Blank/CMakeLists.txt
#add_executable(Pico
#${CMAKE_SOURCE_DIR}/src/main.cpp)
#link_libraries(${PROJECT_NAME})
```

### `README.md`

Optional board-specific notes: pinout diagrams, special build instructions, or hardware caveats.

## Key configuration options

### GPIO pin assignments

Map GPIO pins to button actions using `GpioAction` enum values. Pins not assigned to buttons can be reserved for add-ons using `ASSIGNED_TO_ADDON`.

The Pico config shows a full button layout:

```c
// configs/Pico/BoardConfig.h (pin mapping excerpt)
//                                                  // GP2040 | Xinput | Switch  | PS3/4/5  | Dinput | Arcade |
#define GPIO_PIN_02 GpioAction::BUTTON_PRESS_UP     // UP     | UP     | UP      | UP       | UP     | UP     |
#define GPIO_PIN_03 GpioAction::BUTTON_PRESS_DOWN   // DOWN   | DOWN   | DOWN    | DOWN     | DOWN   | DOWN   |
#define GPIO_PIN_04 GpioAction::BUTTON_PRESS_RIGHT  // RIGHT  | RIGHT  | RIGHT   | RIGHT    | RIGHT  | RIGHT  |
#define GPIO_PIN_05 GpioAction::BUTTON_PRESS_LEFT   // LEFT   | LEFT   | LEFT    | LEFT     | LEFT   | LEFT   |
#define GPIO_PIN_06 GpioAction::BUTTON_PRESS_B1     // B1     | A      | B       | Cross    | 2      | K1     |
#define GPIO_PIN_07 GpioAction::BUTTON_PRESS_B2     // B2     | B      | A       | Circle   | 3      | K2     |
#define GPIO_PIN_08 GpioAction::BUTTON_PRESS_R2     // R2     | RT     | ZR      | R2       | 8      | K3     |
#define GPIO_PIN_09 GpioAction::BUTTON_PRESS_L2     // L2     | LT     | ZL      | L2       | 7      | K4     |
#define GPIO_PIN_10 GpioAction::BUTTON_PRESS_B3     // B3     | X      | Y       | Square   | 1      | P1     |
#define GPIO_PIN_11 GpioAction::BUTTON_PRESS_B4     // B4     | Y      | X       | Triangle | 4      | P2     |
#define GPIO_PIN_12 GpioAction::BUTTON_PRESS_R1     // R1     | RB     | R       | R1       | 6      | P3     |
#define GPIO_PIN_13 GpioAction::BUTTON_PRESS_L1     // L1     | LB     | L       | L1       | 5      | P4     |
#define GPIO_PIN_16 GpioAction::BUTTON_PRESS_S1     // S1     | Back   | Minus   | Select   | 9      | Coin   |
#define GPIO_PIN_17 GpioAction::BUTTON_PRESS_S2     // S2     | Start  | Plus    | Start    | 10     | Start  |
#define GPIO_PIN_18 GpioAction::BUTTON_PRESS_L3     // L3     | LS     | LS      | L3       | 11     | LS     |
#define GPIO_PIN_19 GpioAction::BUTTON_PRESS_R3     // R3     | RS     | RS      | R3       | 12     | RS     |
#define GPIO_PIN_20 GpioAction::BUTTON_PRESS_A1     // A1     | Guide  | Home    | PS       | 13     | ~      |
#define GPIO_PIN_21 GpioAction::BUTTON_PRESS_A2     // A2     | ~      | Capture | ~        | 14     | ~      |

// Pins reserved for add-ons
#define GPIO_PIN_00 GpioAction::ASSIGNED_TO_ADDON
#define GPIO_PIN_01 GpioAction::ASSIGNED_TO_ADDON
#define GPIO_PIN_15 GpioAction::ASSIGNED_TO_ADDON
#define GPIO_PIN_28 GpioAction::ASSIGNED_TO_ADDON
```

### I2C display configuration

To enable an SSD1306/SH1106/SH1107 display over I2C, define the following:

```c
#define HAS_I2C_DISPLAY 1
#define I2C0_ENABLED    1
#define I2C0_PIN_SDA    0   // GPIO pin for SDA
#define I2C0_PIN_SCL    1   // GPIO pin for SCL

// Default button layout on the display
#define BUTTON_LAYOUT       BUTTON_LAYOUT_STICKLESS
#define BUTTON_LAYOUT_RIGHT BUTTON_LAYOUT_STICKLESSB
```

### RGB LED configuration

For WS2812B (NeoPixel) LEDs, define the data pin, brightness, and per-button LED indices:

```c
#define BOARD_LEDS_PIN        28    // GPIO pin for LED data
#define LED_BRIGHTNESS_MAXIMUM 100  // 0–255
#define LED_BRIGHTNESS_STEPS   5
#define LED_FORMAT             LED_FORMAT_GRB
#define LEDS_PER_PIXEL         1

// LED index per button (0-based, order in the LED strip)
#define LEDS_DPAD_LEFT   0
#define LEDS_DPAD_DOWN   1
#define LEDS_DPAD_RIGHT  2
#define LEDS_DPAD_UP     3
#define LEDS_BUTTON_B3   4
#define LEDS_BUTTON_B4   5
#define LEDS_BUTTON_R1   6
#define LEDS_BUTTON_L1   7
#define LEDS_BUTTON_B1   8
#define LEDS_BUTTON_B2   9
#define LEDS_BUTTON_R2   10
#define LEDS_BUTTON_L2   11
```

### Turbo LED

```c
#define TURBO_ENABLED  1
#define GPIO_PIN_14    GpioAction::BUTTON_PRESS_TURBO
#define TURBO_LED_PIN  15
```

## Creating a new board config

<Steps>
  <Step title="Copy the Blank config as a starting point">
    Create a new directory under `configs/` named after your board, then copy the files from `configs/Blank/`:

    ```bash
    cp -r configs/Blank configs/MyBoard
    ```

    Rename the directory to match exactly what you will use for `GP2040_BOARDCONFIG`. The name is case-sensitive.
  </Step>

  <Step title="Edit BoardConfig.h">
    Open `configs/MyBoard/BoardConfig.h` and make the following changes:

    1. Update the include guard and label:

    ```c
    #ifndef MYBOARD_BOARD_CONFIG_H_
    #define MYBOARD_BOARD_CONFIG_H_

    #define BOARD_CONFIG_LABEL "My Board"
    ```

    2. Add GPIO pin mappings for each button on your hardware. Use the `GPIO_PIN_XX` macros where `XX` is the GPIO number (0–28 on RP2040):

    ```c
    #define GPIO_PIN_02 GpioAction::BUTTON_PRESS_UP
    #define GPIO_PIN_03 GpioAction::BUTTON_PRESS_DOWN
    // ... add all buttons
    ```

    3. Enable and configure any peripherals your board has (display, LEDs, etc.).
  </Step>

  <Step title="Create a board CMake file (optional)">
    If your board requires a non-standard `PICO_BOARD` target (for example, `pico2` instead of `pico`), create a `configs/MyBoard/MyBoard.cmake` file:

    ```cmake
    # configs/MyBoard/MyBoard.cmake
    set(PICO_BOARD pico2)
    set(PICO_PLATFORM rp2350)
    ```

    The root `CMakeLists.txt` includes this file automatically if it exists at `configs/<GP2040_BOARDCONFIG>/<GP2040_BOARDCONFIG>.cmake`.
  </Step>

  <Step title="Build and test your config">
    Configure and build targeting your new board:

    ```bash
    mkdir build && cd build
    GP2040_BOARDCONFIG=MyBoard cmake -DCMAKE_BUILD_TYPE=Release ..
    cmake --build . --parallel $(nproc)
    ```

    The output file will be named:

    ```
    build/GP2040-CE_<version>_MyBoard.uf2
    ```

    Flash it to your board and verify that all buttons register correctly using the web configurator's pin viewer.
  </Step>
</Steps>

<Warning>
  GPIO numbers in `BoardConfig.h` are RP2040/RP2350 GPIO numbers, not physical pin numbers on the Pico header. GPIO 0 corresponds to physical pin 1, GPIO 1 to physical pin 2, and so on — consult your board's datasheet.
</Warning>

## Submitting your board config

Once your board config is working, consider submitting it to the project so other users of the same hardware can benefit. See [Contributing](/development/contributing) for the pull request process.
